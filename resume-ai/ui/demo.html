Semantic Similarity Strategy

The Kafka ↔ Kinesis problem is solved with a three-layer approach:

Layer 1: Grounding alias map
Deterministic tech equivalences injected into system prompt. Prevents hallucinated mappings.
Layer 2: LLM semantic reasoning
Claude reads JD + resume and scores similarity using the alias map as grounding context.
Layer 3: Explainability
Every score includes direct resume quotes as evidence — human reviewer can verify reasoning.
Scale: 10,000+ resumes/day
🔢 Cost optimisation: Heuristic pre-filter removes bottom 60% before any LLM call
⚡ Throughput: asyncio + 20 workers → ~3.4 resumes/sec = 12,000/hr
🏗 Architecture: SQS queue → stateless worker pods → PostgreSQL/Redis
💰 Estimated cost: ~$10/day for 10k resumes
